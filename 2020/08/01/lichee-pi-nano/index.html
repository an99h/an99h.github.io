lichee_pi_nano

2020-08-01 Comments

荔枝派nano SPI Flash启动Linux

  • CPU:全志 F1C100s
  • Flash型号：xt25f128b
  • Flash容量：16M
  • 烧写Flash工具：sunxi-tools
  • ToolChain：gcc-linaro-7.2.1-2017.11-x86_64_arm-linux-gnueabi
  • U-boot：2018.01
  • Linux内核：4.15
  • 编译环境：Ubuntu 16.04

准备工作

  • U-BOOT
  • Linux
  • ToolChain
  • sunxi-tools
  • 进入spl模式两种方式
    1. 上电时拉低SPI flash 的 CS引脚
    2. u-boot下：sf probe 0;sf erase 0 0x100000;reset

U-BOOT

由于官方板子用的Flash型号是w25q128，我买的板子Flash型号是xt25f128b，经过查找资料得知这两块Flash指令几乎完全一样，但是ID不同u-boot不能识别xt25f128b，需要自己修改u-boot spi代码。

  • u-boot/drivers/mtd/spi/spi_flash_ids.c
  • 编译u-boot
  • 将u-boot烧入flash启动后
    sf probe 0即可看到flash挂载成功
  • 修改之前
  • 配置u-boot menuconfig
  • 生成SPI Falsh可以用的bin文件
    u-boot-sunxi-with-spl.bin

Linux

  • 修改make menuconfig 适配SPI Flash
  • 修改spi驱动适配xt25f128b Flash
    linux/arch/arm/boot/dts/suniv-f1c100s-licheepi-nano.dts

/linux/drivers/mtd/devies/m25p80.c最后增加xt25f128b定义

/linux/drivers/mtd/spi-nor/flash_info spi_nor_ids[]中修改

  • 编译zImage dtb

编译rootfs

  • 获取buildroot源码

  • 下载依赖工具包
    https://pan.baidu.com/s/1murPEvc1GuchQ2d9QaEM2Q 提取码: 3rcq
    下载完之后，解压后复制到”buildroot-2017.08/dl/“下，没有dl文件夹可自行创建。
    也可以不下载此工具包，但是buildroot自动更具需求下载这些工具包也许很慢

  • 编译rootfs

  • 编译完生成rootfs.tar在 output/image目录下

打包SPI Flash bin文件

将生成的以下文件找到放到一个目录下

  • suniv-f1c100s-licheepi-nano.dtb
  • u-boot-sunxi-with-spl.bin
  • zImage
  • rootfs.tar

1.先生成jffs2.img文件,不然每次打包会消耗很长时间

2.打包脚本

烧写SPI Flash

  1. 先进入SPL模式
  2. sunxi-fel -p spiflash-write 0 flashimg.bin

上电启动
