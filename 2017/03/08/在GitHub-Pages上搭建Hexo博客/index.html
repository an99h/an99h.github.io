在GitHub Pages上搭建Hexo博客

Mar 08, 2017教程&总结Golpes

前言

作为前端小白只能在前人基础上搭建网站创建自己博客。为什么会选择在GitHub Pages上搭建博客？因为空间免费，操作简单，用来做搭建一个博客再好不过了。在GitHub Pages创建博客可以认为是用户编写，托管在GitHub上的静态网页。
Hexo是一个简单、快速、强大的博客发布工具，支持Markdown格式。

前期准备

注册GitHub账号

http://www.github.com/ 注册过程比较简单
在GitHub上新建xxx.github.io仓库（xxx为自己用户名）

安装git

安装hexo

购买域名

这一步可选，去阿里云买个不常见的顶级域名很便宜的，我的.space域名去年买的，首年4元，又续了两年,也不算贵总共26元。

本地创建hexo后台

1.创建hexo博客文件夹

Hexo会自动在目标文件夹建立网站所需要的所有文件

2.编译当前博客

3.启动本地server，浏览器输入localhost:4000

如果进入博客首页，可以进入下一项设置。

配置hexo _config.yml

1.最后几行配置如下，==注意：后面要有空格==

2.首先配置本地git账户，下次再介绍。

3.推送本地静态博客到GitHub Pages。

4.在浏览器输入xxx.github.io,不出问题应该可以打开和之前本地一样的界面

绑定自己域名

  1. 在阿里云域名解析，添加CNAME记录,记录值为xxx.github.io.
  2. 在本地hexo/source/目录下新建CNAME文件没有后缀名，内容为自己注册的域名。
  3. 再次编译自己博客

5.推送完成用自己域名试试，应该大功告成了。
